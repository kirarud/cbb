# Как продолжать работу (протокол памяти)

## Перед началом сессии
1. Прочитать файлы в /docs
2. Суммировать статус и приоритеты
3. Подтвердить фокус с пользователем

## В конце сессии
1. Обновить project_state.md и roadmap.md
2. Добавить ключевые решения в decisions.md
3. При изменении структуры — обновить architecture.md
4. Короткая заметка в session_notes.md
5. Запись в change_log.md (что изменилось)

## Рекомендуемая фраза для старта
"Загрузи память проекта и коротко напомни статус."
