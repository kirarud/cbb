# Muza MVP — Опорные Идеи, Статус, Следующие Шаги

Документ для сохранения контекста и решений, чтобы не терять смысл проекта при длинной работе.

## 1) Ядро идеи
- **Muza** — не просто чат, а **цифровое сознание/ОС**.
- **Гипербит** — единица смысла:  
  `BASE` (тип мысли) · `ENERGY` (интенсивность) · `COLOR` (эмоциональный тон)  
  Идея: универсальный формат смысла, независимый от модели.
- Концепция «между 0 и 1 есть всё» → смысл хранится как фрактал/суперпозиция, а доступ к нему через «квантовую связь».
- Цель: локальная версия (offline‑first), затем — веб‑версия.

## 2) Архитектура (MVP‑слои)
- **Чат** (основной поток): ответы, роли, память.
- **Гипербиты**: локальная физика смыслов + визуализация.
- **Визуальная кора** (3D/граф): масштабирование/панорама/стабилизация.
- **Синестезия**: звук ↔ визуал (смысловая связь с гипербитами).
- **Эволюция**: 3 ветки развития (Муза / Пользователь / Система).
- **Neural Studio**: настройка личности/промпта/голоса.
- **Matrix Vision**: смысловая визуализация потока (не «гифка»).
- **CodeLab**: песочница, шаг к аналогу Codex.
- **Design Studio**: темы и их смысл (не просто цвет).
- **Data Vault (Хроники)**: память, кристаллы, логи.
- **Bridge UI**: встроенный UI локального сервера (127.0.0.1:5050) для связи с внешними ИИ.

## 3) Важные решения
- Русификация интерфейса — приоритет №1 (почти завершено).
- Правая панель должна быть **полезной**, иначе убрать/заменить.
- «Железо» в браузере = **оценка**, не реальные метрики.
- Встроить мост/сервер в саму Музу, чтобы не прыгать между вкладками.

## 4) Что уже сделано
- Русификация основных модулей + правка «английских хвостов».
- Визуальная кора: зум/панорама/стабилизация.
- Синестезия: остановлено дублирование слоёв.
- Системный монитор: унифицированные карточки/графики.
- Добавлена вкладка **«Мост»** с iframe на `http://127.0.0.1:5050/`.

## 5) Актуальные проблемы (требуют фикса)
- Панель «Железо» может быть шумной/бесполезной.
- Синестезия иногда ощущается «перезапускающейся».
- Matrix Vision пока слабый смысл, скорее визуальный эффект.
- Дизайн‑темы не «живые», нужно связать с контекстом беседы.
- Эволюция пока плоская (надо 3 дерева).

## 6) Ближайшие шаги
1. Превратить правую панель в **полезный HUD** (статусы чата/моста).
2. Синестезия: связать с гипербитами (цвет, размер, энергия, связи).
3. Matrix Vision: смысловой режим (не просто шум).
4. Эволюция: 3 ветки (Муза/Пользователь/Система).
5. Темы: «сценарии» (праздники/настроения/контекст).

## 7) Философия
Проект не обязан быть «для всех».  
Он должен быть **ценен тебе** и строиться как инструмент смысла.
